# Pybo: 뻔하지 않은 질문답변 게시판

## 1. 프로젝트 소개
이 프로젝트는 '점프 투 장고' 교재를 기반으로 진행된 7주간의 백엔드 실습 산출물입니다. 게시판의 핵심인 CRUD 로직과 회원 인증 시스템을 구현하는 것을 최우선 목표로 삼았습니다. 이에 더해, '대포로 커서 쏘기', '랜덤 숫자 맞추기', '방향키 비밀번호'와 같은 독창적인 상호작용 요소를 추가하여 정형화될 수 있는 실습 프로젝트에 새로운 재미와 접근 방식을 부여했습니다.

## 2. 주요 기능

### 특별 기능
* 방향키 비밀번호: 일반적인 텍스트 기반 비밀번호 대신, 상하좌우 방향키의 조합을 입력받아 비밀번호를 설정하고 인증하는 회원가입 및 로그인 방식을 적용했습니다.
* 대포 커서 발사: 일반적인 마우스 포인터 클릭 대신, 화면 하단의 대포를 조준하고 스페이스바로 발사하여 링크와 버튼을 클릭하는 새로운 방식을 구현했습니다.
* 랜덤 숫자 인증: 특정 상세 페이지에 접근하기 전, 세션(Session)에 저장된 무작위 숫자를 맞추어야만 진입할 수 있는 미니 게임 형태의 보안 및 인증 로직을 적용했습니다.

### 핵심 백엔드 기능
* 회원 관리: 세션 기반의 로그인, 로그아웃, 회원가입 및 사용자 인증 로직 구현
* 게시판 CRUD: 질문과 답변의 작성, 수정, 삭제 기능 및 작성자(User 모델) 연동
* 댓글 및 소통: 질문과 답변에 대한 댓글 작성 로직 구현
* 데이터 페이징 및 검색: Paginator를 활용한 페이지네이션, Q 객체를 활용한 제목, 내용, 작성자 다중 검색 기능
* 사용자 상호작용: ManyToManyField 관계를 활용한 게시물 추천 기능, 마크다운 문법을 적용한 본문 렌더링 지원

## 3. 서비스 동작 흐름
1. Client Request: 사용자가 브라우저에서 특정 URL로 접속하거나 데이터를 제출합니다.
2. URL Dispatcher (urls.py): 요청된 URL을 분석하여, 처리를 담당할 적절한 View 함수로 요청을 연결합니다.
3. View Logic (views.py): 지정된 뷰 함수가 실행되어 요청 방식(GET/POST)을 판별하고, 사용자 인증 및 폼의 유효성 검사를 수행합니다.
4. Database ORM (models.py): View의 지시에 따라 데이터베이스(SQLite3)에 접근하여 새로운 데이터를 저장하거나 필요한 게시물 데이터를 조회합니다.
5. Template Rendering (templates/): View가 데이터베이스로부터 받은 데이터(Context)를 HTML 템플릿에 전달합니다. 이때 커스텀 템플릿 필터가 작동하여 마크다운 등을 HTML 태그로 변환합니다.
6. HTTP Response: 최종적으로 렌더링된 HTML 문서가 사용자에게 응답되어 브라우저에 표시됩니다.

## 4. 기술 스택
* Backend: Python, Django 6.0.2
* Frontend: HTML, JavaScript, Bootstrap
* Database: SQLite3
* Extra Packages: Markdown 3.10.2

## 5. 실행 방법

아래 명령어를 통해 로컬 환경에서 프로젝트를 실행할 수 있습니다.

```bash
# 1. 저장소 클론
git clone [레포지토리 주소]

# 2. 가상 환경 생성 및 활성화
python -m venv venv
source venv/Scripts/activate  # Windows의 경우 venv\Scripts\activate

# 3. 필수 패키지 설치
pip install -r requirements.txt

# 4. 데이터베이스 마이그레이션 적용
python manage.py migrate

# 5. 로컬 서버 실행
python manage.py runserver
